🎉 正しいOAuth2クライアント設定手順

✅ Google Cloud Console設定:

  1. Google Cloud Console → APIs & Services → 認証情報
  2. 「+ 認証情報を作成」→「OAuth 2.0 クライアント ID」
  3. アプリケーションの種類: 「Webアプリケーション」を選択
  4. 名前: 「CLASSY Business Management」
  5. 承認済みのリダイレクトURI: 現在のページURL
  6. 「作成」をクリック
  7. クライアントIDとクライアントシークレットをコピー

🔑 認証設定手順

  1. 上記で取得した「クライアントシークレット」を入力
  2. 必要なスコープを選択
  3. 「PKCE認証でログイン」をクリック
  4. Googleアカウントでログイン
  5. 権限を許可
  6. 自動的にカレンダー・タスク同期開始

🚨 エラー解決方法

「invalid_client」エラーの場合:

  • クライアントIDが正しくコピーされているか確認
  • クライアントシークレットが正しくコピーされているか確認
  • リダイレクトURIが完全一致で設定されているか確認
  • Google Cloud ConsoleでOAuth同意画面が設定されているか確認
  • Calendar APIとTasks APIが有効化されているか確認
